# 17 — Keyword Consistency and Content Fit

## Goal

Evaluate whether a page matches its intended search queries and user intent without using outdated keyword-density logic.

## Inputs

Best with:

- target keywords;
- priority pages;
- Search Console query export;
- competitor pages;
- current title/H1/body text;
- business goals.

If no target keywords are provided, infer likely topics from page content and label the analysis as inferred.

## Checks

For each important page:

- title matches the page topic;
- H1 matches or complements the title;
- primary topic appears naturally in first-screen content;
- meta description supports the same promise;
- H2/H3 cover expected subtopics;
- visible text answers likely user questions;
- internal anchors use relevant descriptive language;
- image alt and captions support the topic when appropriate;
- page does not target too many unrelated intents;
- page does not rely on keyword stuffing.

## Entity/topic coverage

Evaluate whether content includes relevant entities and concepts.

Example for a web studio service page:

- website development;
- landing page / corporate site / ecommerce;
- UX/UI;
- frontend/backend;
- CMS/admin panel;
- integrations;
- analytics;
- performance;
- launch/support;
- pricing/timelines;
- portfolio/cases;
- region/service area if relevant.

## SERP snippet preview

For priority pages show:

```md
Title:
URL:
Description:
Intent match:
Snippet risk:
Recommendation:
```

## Warnings

Do not recommend:

- repeating exact keywords mechanically;
- stuffing keywords into every heading;
- adding irrelevant blocks just to increase word count;
- hiding keyword text.

## Reporting

Flag as P1/P2 when a high-value page targets the wrong intent or lacks the vocabulary needed to be understood.
